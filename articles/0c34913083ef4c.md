### 記憶について
claude codeではできたことでgemini cliではできないこととして、
geminiは会話のログを勝手に保存したり、1度抜けたセッションを再開したりすることができないぽい。
なので意図的にセッション内で会話を覚えるように命令する必要があるらしい。

ただ、これは自己管理の範疇な気がしているのでおとなしく自分でメモを取ることにする。

## prompt
geminiコマンドでエディターを展開しなくても、簡単な命令だったらこれでいいかも

```bash
❯ gemini --prompt "1 たす 1 は？"

2です。
```

## catの結果を送信する
これのいいところはgemini cliの画面内でこのファイルを読んでこの質問に答えてください。と手順を踏んで質問したり、その間にgeminiからの権限リクエストの対応をしないといけないのをパスできそうな点です。
あまりに長文だと読み取ってくれなさそうですが、前提知識不要な簡単なファイルを読んでもらう分には問題ないかと。

```bash
# ファイルの内容
❯ cat tashizan.txt

1 たす 1 は？
```

```bash
~
❯ cat tashizan.txt| gemini --prompt "このファイルはどんな言語で書かれている？"

1+1は2です。どのファイルについて知りたいですか？

~ took 13s
❯ cat tashizan.txt| gemini --prompt "このファイルに書かれている言語がどこの国の言語か知りたいです。"

その言語は日本語です。
1たす1は2です。
```

### macにおけるデメリット
個人的にメモを取る習慣を心がけているが、catコマンドでmacのメモ機能は呼び出すことができない(場所が複雑すぎる)
ので、AIを今後活用していく上では、macのメモ以外の場所でメモをとった方が良いのではという結論に至った。

## mac今後AI開発していくためのドキュメントのあり方
前述した通り、macのメモ機能はAI開発するにおいては完全にいらないと思った。
obsidianや適当なメモディレクトリ作ってテキストメモをとるのが最善かもしれない。
また、適切にAIにいい感じに読み込ませるためには、自分の思考ログやAIから得た回答による知見みたいなものを自分なりに都度、収集していく必要があるのではないか・・・
そのためのCLAUDE.mdであったりGEMINI.mdであると思うのだが、その1ファイルにまとめるわけではなく、分散してカテゴリごとのmdでまとめて、それをAIに読み込ませるといいなと思った。
